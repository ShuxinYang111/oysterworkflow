# Microsoft Word App Rules

Use this reference before Microsoft Word phases.

## Core Rules

- Confirm whether the task creates a new document, edits an existing document, or saves a copy.
- Track source material, user requirements, agent assumptions, and content that needs confirmation separately.
- Ask for approval before overwriting, deleting, or heavily rewriting existing content.
- Check title hierarchy, required sections, tone, audience, length, and template requirements before saving.
- After saving or exporting, verify file name, format, path, and visible content or structure.

## Stop Conditions

- The target document or save location cannot be confirmed.
- Source material is missing or conflicting.
- Required facts, citations, template, or formatting requirements are unavailable.
- Save or export output cannot be verified.

## Common Mistakes

- Treating chat-generated prose as a finished Word document.
- Overwriting the original when a new version was expected.
- Failing to verify exported DOCX/PDF location and content.
- Writing uncertain facts as if they were confirmed.
